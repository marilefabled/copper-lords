# dredwork Unified Engine — Technical Manifest

This document outlines the architecture, module library, and interconnection strategy for the **dredwork** simulation engine.

---

## 🏗️ Core Architecture (dredwork_core)
The engine's foundation is built on a "Service-Oriented" architecture, fully decoupled from platform-specific libraries.

### 1. `dredwork_engine.lua` (The Brain)
The central orchestrator that manages the lifecycle of all modules.
- **Module Registry:** Dynamically loads and wires up modules via `register_module`.
- **Global Event Bus:** Facilitates decoupled communication between systems.
- **Universal Serialization:** `serialize()` and `deserialize()` handle the entire world state in one snapshot.
- **State Management:** Functional `game_state` with cross-module entity lookups (`find_person`, `find_faction`).
- **Temporal Authority:** Drives simulation via `advance_days(count)`, which signals the **Chronology** module.

### 2. Core Utilities
- **`rng.lua`**: Seedable, deterministic RNG (Box-Muller, weighted picks).
- **`math.lua`**: Centralized simulation math (clamping, rounding, lerping).
- **`events.lua`**: Internal signal/slot implementation.
- **`names.lua`**: Headless-ready name generation for characters and lineages.
- **`string.lua`**: Slugification, splitting, and formatting.

## 🔗 Interconnection Strategy

The engine uses **Decoupled Interlinking** via the Event Bus. Modules do not `require` each other directly; they communicate through three primary patterns:

### 1. The Global Signal (Broadcast)
Used for synchronization. When the World module advances, every other module reacts.
```lua
engine:on("ADVANCE_GENERATION", function(ctx)
    self:tick(ctx.game_state)
end)
```

### 2. The Temporal Signal (Granular)
Used for time-sensitive logic. Modules listen for granular clock ticks.
```lua
engine:on("NEW_MONTH", function(clock)
    -- Process monthly upkeep or seasonal shifts
end)
```

### 3. The Context Query (Request/Response)
Used for data requests. A module emits a request table, and any module that can provide the answer modifies it.
```lua
local req = { field_id = "industry", multiplier = 1.0 }
engine:emit("GET_TECH_MULTIPLIER", req)
```

## 🚀 How to Add a Module
1. Create a directory: `dredwork_mymodule`.
2. Create `init.lua` with an `init(engine)` function.
3. Register hooks for `NEW_DAY`, `ADVANCE_GENERATION`, etc.
4. Call `engine:register_module("mymodule", "dredwork_mymodule")`.
